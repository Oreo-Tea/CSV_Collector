# PLC Data Collector 基本設計

## 1. 目的

本アプリケーションは、複数 PLC からの CSV データ受信と Modbus/TCP ポーリングを行い、指定フォルダへデータを保存する WinForms アプリケーションです。

## 1.1 前提条件

- Windows 環境での実行を想定
- .NET 10 SDK / Windows Desktop Runtime が利用可能
- PLC は Modbus/TCP に対応し、CSV データ送信先として本アプリの IP/ポートを受け入れる
- 保存先フォルダへ書き込み権限がある

## 2. システム構成

- UI 層
  - `MainForm.cs`: 起動・停止、設定画面起動、サービスのオーケストレーション
  - `SettingsForm.cs`: 設定値の表示・編集
- 設定モデル
  - `Models/AppConfig.cs`: 設定パラメータを保持
- サービス層
  - `Services/CsvListener.cs`: TCP 受信サーバー、CSV 受信イベント発行
  - `Services/FileStorageService.cs`: 受信 CSV の保存、保存間隔制御、古いデータ削除
  - `Services/ModbusService.cs`: PLC への Modbus/TCP ポーリング、状態通知
  - `Services/ModbusStorageService.cs`: Modbus 読み取り結果の CSV 保存
  - `Services/LoggingService.cs`: 日次ログ出力、ログローテーション、ログ保持

### 2.1 システム構成図

```
+----------------------+        +------------------------+
|  PLC (複数)          |        |  PLC Data Collector    |
|  - Modbus/TCP        |<------>|  WinForms App          |
|  - CSV 出力先        |  TCP   |  - CsvListener         |
+----------------------+        |  - FileStorageService  |
                                |  - ModbusService       |
                                |  - ModbusStorageService|
                                |  - LoggingService      |
                                +-----------+------------+
                                            |
                                            v
                                +------------------------+
                                |  保存フォルダ           |
                                |  - yyyy/MM/dd/CSV      |
                                |  - Logs/               |
                                +------------------------+
```

### 2.2 主要コンポーネント一覧

| コンポーネント | 役割 | 主な出力先 / 連携先 |
|---|---|---|
| `MainForm` | ユーザー操作とサービス制御 | `CsvListener`, `FileStorageService`, `ModbusService`, `LoggingService` |
| `SettingsForm` | 設定編集と保存 | `AppConfig` |
| `AppConfig` | 設定値保持と読み書き | `appsettings.json`, 各サービス |
| `CsvListener` | TCP で CSV 受信 | `CsvReceived` イベント -> `FileStorageService` |
| `FileStorageService` | CSV 保存、保持制御 | データフォルダ, `ModbusStorageService` ではないが保存制御の参考 |
| `ModbusService` | PLC の Modbus ポーリング | `StatusUpdated` イベント -> `ModbusStorageService` |
| `ModbusStorageService` | Modbus 読み取り結果の CSV 保存 | データフォルダ |
| `LoggingService` | アプリログ出力・ローテーション | `Logs` フォルダ |

## 3. 主要データフロー

1. `Program.Main` が `appsettings.json` を読み込み、`AppConfig` を初期化
2. `MainForm` を起動し、現在の PLC IP リストを表示
3. [Settings] で `AppConfig` を編集し、保存時に `appsettings.json` に書き込み
4. [Start] クリックで以下を開始
   - `LoggingService` を初期化
   - `CsvListener` を開始し TCP ポートで CSV データを受信
   - `FileStorageService` を開始し、受信 CSV を一定間隔でファイル保存
   - `ModbusService` を開始し、PLC を定期的にポーリング
   - `ModbusStorageService` を初期化し、Modbus 読み取り結果を保存
5. CSV を受信すると `CsvListener` が `CsvReceived` イベントを発火
6. `MainForm` が受信イベントを受け取り、`FileStorageService.ReceiveCsv` を呼び出し最新データを保持
7. タイマー周期で `FileStorageService.FlushAsync` が最新 CSV を日付別フォルダへ保存
8. `ModbusService` が各 PLC に接続して保持レジスタを読み取り、`StatusUpdated` イベントを発火
9. `MainForm` がイベントを受け取り、`ModbusStorageService.SaveAsync` で Modbus データを CSV 保存

## 4. 設計要素と責務

### 4.1 `AppConfig`
- CSV リスナーのポート
- 保存間隔、保持年数、データフォルダ
- ログレベル、ログ保持日数、ログ最大サイズ
- Modbus 関連設定（スレーブ ID、読み取り開始アドレス、読み取り件数、ポーリング間隔）
- Modbus データ保存 ON/OFF

### 4.2 `CsvListener`
- `TcpListener` で `IPAddress.Any` を監視
- 接続ごとに受信データを集約し、UTF-8 の文字列に変換
- 受信 CSV を `CsvReceived` イベントとして通知

### 4.3 `FileStorageService`
- `ConcurrentDictionary<string, string>` で IP ごとの最新 CSV をキャッシュ
- タイマーで定期的に `FlushAsync` を実行し、1 日単位フォルダへ保存
- 保存後、設定保持年数を超えた古い日付フォルダを削除

### 4.4 `ModbusService`
- `TcpClient` で PLC の Modbus/TCP ポート(502)に接続
- 接続タイムアウト 3 秒
- `ModbusIpMaster.ReadHoldingRegisters` で保持レジスタを読み取り
- 読み取り結果を `StatusUpdated` イベントで通知

### 4.5 `ModbusStorageService`
- Modbus 読み取り結果を CSV に変換して保存
- `SaveModbusToCsv` が `true` の場合にのみ保存

### 4.6 `LoggingService`
- `DataFolder/Logs` に日別ログファイルを作成
- 設定レベル以上のログを出力
- ファイルサイズ上限に達したらローテーション
- 古いログをログ保持日数で削除

## 5. UI と操作

- `MainForm`
  - `Settings` ボタンで設定画面表示
  - `Start` ボタンで各サービス開始
  - `Stop` ボタンで各サービス停止
  - 終了時に `Stop` を呼び出してリソースを解放
- `SettingsForm`
  - PLC IP を複数行で入力
  - ポート、保存間隔、保持年数、データフォルダを編集
  - 保存時にフォルダを作成し、設定を反映

## 6. 改善案 / 今後の拡張

- 例外ハンドリングを詳細化し、UI へエラー表示
- `CancellationToken` を全サービスで共通化
- 依存性注入（DI）によるサービス分離
- 設定値バリデーションの追加
- CSV 受信時のプロトコル検証（ヘッダー/改行対応）
- Modbus 双方向制御や書き込み対応
- データベース保存サポート（SQLite など）
- ステータス表示・監視 UI の追加

## 7. 現状の制約

- CSV 受信処理はシンプルな TCP 受信で、改行/チャンク分割を明示的に扱っていない
- ログ出力と保存処理で例外を握りつぶす箇所がある
- UI は最小構成で、進行状況やエラーは画面に表示されない
- Modbus は常にポーリングのみで、PLC からのプッシュ通知には対応しない
