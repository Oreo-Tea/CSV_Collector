# PLC Data Collector 詳細設計

## 1. 前提

本アプリケーションは Windows Forms ベースのクライアントアプリとして動作し、複数 PLC からの CSV 受信と Modbus/TCP ポーリング結果を保存する。

- 対象プラットフォーム: Windows
- フレームワーク: .NET 10.0 (Windows デスクトップ)
- 通信: TCP/IP、Modbus/TCP
- 保存: ファイルシステムへの日次保存

## 2. コンポーネント構成

### 2.1 UI 層

- `MainForm.cs`
  - アプリ起動後のメイン画面
  - `Settings` ボタンで設定フォーム表示
  - `Start`/`Stop` でサービス制御
  - `CsvListener` と `ModbusService` のイベントを受信し、保存処理を起動

- `SettingsForm.cs`
  - 設定パラメータ編集画面
  - PLC IP リスト、CSV 受信ポート、保存間隔、保持年数、保存フォルダを入力
  - 設定を `AppConfig` に反映し `appsettings.json` に保存

### 2.2 モデル層

- `Models/AppConfig.cs`
  - 各種設定値を保持するデータクラス
  - `PlcIps`, `CsvListenerPort`, `SaveIntervalSeconds`, `RetentionYears`, `DataFolder`
  - `LogLevel`, `LogRetentionDays`, `LogMaxFileSizeBytes`
  - `ModbusSlaveId`, `ModbusReadStart`, `ModbusReadCount`, `ModbusPollIntervalSeconds`
  - `SaveModbusToCsv`

### 2.3 サービス層

- `Services/LoggingService.cs`
  - 日別ログファイル生成
  - ログレベルフィルタリング
  - ローテーション（サイズ上限到達時のローテート）
  - 古いログの削除

- `Services/CsvListener.cs`
  - `TcpListener` で指定ポートを監視
  - 受信した TCP データを UTF-8 CSV として変換
  - `CsvReceived` イベントを発行

- `Services/FileStorageService.cs`
  - 受信 CSV の最新データを IP ごとにキャッシュ
  - タイマーで `FlushAsync` を定期実行し、日付別フォルダに保存
  - 保存後、指定年数より古いフォルダを削除

- `Services/ModbusService.cs`
  - 各 PLC の Modbus/TCP ポートへ接続
  - 3 秒の接続タイムアウト
  - `ModbusIpMaster.ReadHoldingRegisters` で保持レジスタ取得
  - `StatusUpdated` イベントを発行し、読み取り結果を通知

- `Services/ModbusStorageService.cs`
  - Modbus 読み取り結果を CSV に変換・保存
  - 日別フォルダに `modbus_{ip}_{timestamp}.csv` 形式で出力

### 2.4 UI 遷移図

```
+---------------------+                 +---------------------+
|      MainForm       |                 |   SettingsForm      |
+---------------------+                 +---------------------+
| - Start             |                 | - PLC IPs           |
| - Stop              |                 | - CSV Port          |
| - Settings          |<---[open]---    | - Save              |
| - Status display    |                 | - Cancel            |
+---------------------+                 +---------------------+
        |  ^                                    |
        |  |                                    |
        |  +------------[close]-----------------+
        |
        +------[service control]-------------->
                 CsvListener / ModbusService
```

- `MainForm` は起動後に最初に表示されるメイン画面
- `Settings` ボタンで `SettingsForm` をモーダル表示
- `Save` 時に設定が `AppConfig` に反映され、`MainForm` の次回起動時に読み込まれる

### 2.4.1 画面項目

#### `MainForm`
- `Start` ボタン: CsvListener と ModbusService を開始
- `Stop` ボタン: すべてのサービスを停止
- `Settings` ボタン: `SettingsForm` を表示
- `Status` ラベル/テキスト領域: 現在の稼働状態、エラーメッセージ、受信数を表示
- `LastReceived` / `LastSaved` 表示: 最新の CSV 受信時刻や保存時刻を通知
- `Log` ボタン（任意）: ログフォルダを開く動線

#### `SettingsForm`
- `PLC IPs` 入力エリア: 複数 PLC IP を複数行で入力
- `CSV 受信ポート` 入力欄: TcpListener の待ち受けポート
- `保存間隔` 入力欄: CSV フラッシュ周期（秒）
- `保持年数` 入力欄: 過去データの削除基準
- `保存フォルダ` 入力欄: データ保存先ルート
- `Modbus スレーブ ID` 入力欄: Modbus 参照先スレーブ番号
- `Modbus 読み取り開始アドレス` 入力欄: 読み取り開始保持レジスタアドレス
- `Modbus 読み取り件数` 入力欄: 読み取りレジスタ数
- `Modbus ポーリング間隔` 入力欄: ポーリング周期（秒）
- `Modbus CSV 保存` チェックボックス: Modbus 結果を CSV に保存するかどうか
- `Save` / `Cancel` ボタン: 入力内容の確定または破棄

### 2.4.2 エラー遷移

#### `MainForm` のエラー遷移
```
[Start clicked] -> [Start failure]
    -> MainForm: エラーメッセージ表示
    -> LoggingService: エラー出力
    -> 状態を "停止" に戻す

[CsvListener start error] -> MainForm: ポート開放失敗メッセージ
[ModbusService start error] -> MainForm: PLC 接続失敗メッセージ
```

#### `CsvListener` のエラー遷移
```
[Accept/TCP receive error]
    -> CsvListener: 例外キャッチ
    -> LoggingService: エラー出力
    -> MainForm: 短時間的なエラー通知（UI ステータス更新）
    -> 接続をクローズまたは再待機
```

#### `ModbusService` のエラー遷移
```
[PLC 接続失敗 or 読み取り失敗]
    -> ModbusService: 例外キャッチ
    -> LoggingService: エラー出力
    -> MainForm: 該当 PLC の状態を「接続失敗」表示
    -> 次回ポーリングまで待機
```

#### 保存処理のエラー遷移
```
[FileStorageService/ModbusStorageService save error]
    -> サービス: 例外キャッチ
    -> LoggingService: エラー出力
    -> MainForm: 保存失敗をステータスに表示
    -> 次回保存までキャッシュを保持
```

#### `SettingsForm` のエラー遷移
```
[設定保存エラー]
    -> SettingsForm: 入力検証失敗 or ファイル書き込み失敗
    -> エラーをダイアログ表示
    -> LoggingService: エラー出力
    -> 保存処理を中断し、フォームを開いたままにする
```

### 2.5 シーケンス図

#### 2.5.1 CSV 受信シーケンス

```
Client PLC -> MainForm: Start
MainForm -> CsvListener: Start()
Client PLC -> CsvListener: TCP 接続/CSV 送信
CsvListener -> CsvListener: 受信バッファ集約
CsvListener -> MainForm: CsvReceived(event)
MainForm -> FileStorageService: ReceiveCsv(ip, csv)
FileStorageService -> FileStorageService: キャッシュ更新
FileStorageService -> FileStorageService: FlushAsync() [Timer]
FileStorageService -> FileSystem: Save CSV
```

#### 2.5.2 Modbus ポーリングシーケンス

```
MainForm -> ModbusService: Start()
ModbusService -> PLC: TCP/Modbus接続
PLC -> ModbusService: 接続応答
ModbusService -> PLC: ReadHoldingRegisters(start, count)
PLC -> ModbusService: レジスタ値
ModbusService -> MainForm: StatusUpdated(event)
MainForm -> ModbusStorageService: SaveAsync(ip, status)
ModbusStorageService -> FileSystem: Save CSV
```

## 3. データフロー

### 3.1 CSV 受信フロー

1. `MainForm` で `Start` をクリック
2. `CsvListener.Start()` が `TcpListener` を開始
3. TCP 接続受信時、`CsvListener` が受信バイト列を UTF-8 に変換
4. `CsvReceived` イベントを発行
5. `MainForm` が `FileStorageService.ReceiveCsv()` を呼び出し、最新データをキャッシュ
6. タイマー周期で `FileStorageService.FlushAsync()` が実行
7. 日別フォルダに CSV を保存

### 3.2 Modbus ポーリングフロー

1. `MainForm` で `Start` をクリック
2. `ModbusService.Start()` がポーリングループを開始
3. 各 PLC IP に対して TCP 接続を試行
4. 接続成功時に保持レジスタを読み取り
5. `StatusUpdated` イベントを発行
6. `MainForm` が `ModbusStorageService.SaveAsync()` を呼び出し、読み取り結果を CSV 保存

## 4. 設計ルール

- UI は最低限の操作性に留め、サービス責務は分離する
- 設定値は `appsettings.json` に永続化し、起動時にロードする
- 例外はログ出力し、UI を停止させない工夫を行う
- 保存ファイルは日付フォルダで分割し、データ保持期間を設定できる

## 5. ファイル命名規則

- CSV 受信保存: `Data/yyyy-MM-dd/{ip}_{yyyyMMdd_HHmmss}.csv`
- Modbus 保存: `Data/yyyy-MM-dd/modbus_{ip}_{yyyyMMdd_HHmmss}.csv`
- ログ: `Data/Logs/yyyy-MM-dd.log`

## 6. 非機能要件

- 可用性: アプリは停止せずに継続して受信・保存を行う。
- 拡張性: 設定項目は `AppConfig` で拡張可能に設計する。
- 保守性: 各機能をサービス層に分離し、UI とロジックを疎結合にする。
- パフォーマンス: 受信 CSV は最新データのみをキャッシュし、周期的にまとめて保存することでファイル I/O を抑制する。
- ログ: 運用時のイベントはログに残し、上限サイズでローテーションする。

## 7. 既知の制約

- 本実装は Windows Forms 向けのため、macOS や Linux での GUI 実行をサポートしない。
- `NModbus4` の現状パッケージ互換性は .NET 10 で注意が必要で、状況に応じて代替ライブラリに切り替える可能性がある。
- CSV 受信プロトコルは最小実装で、ヘッダーや改行区切り、複数パケット分割の厳密な解析は実装されていない。
- 現在の設計では PLC からのプッシュ通知を処理せず、Modbus はポーリングのみである。

## 12. 保守メモ

- `appsettings.json` の変更は `MainForm` から即時保存されるため、起動中の反映は再起動が必要。
- `FileStorageService` の `FlushAsync` は例外を握りつぶすため、運用時はログを定期確認する。
- `ModbusService` の接続・読み取りタイムアウトは `appsettings.json` で調整できるように拡張可能。

1. UI にステータス表示およびエラーメッセージ欄を追加
2. CSV 受信をストリーム単位で正規化し、改行区切りを厳密に扱う
3. `CancellationToken` を共通化し、シャットダウン時の整然とした停止を実現
4. `NModbus4` の互換性を確認し、必要なら .NET 10 対応の Modbus ライブラリに切り替える
5. SQLite などの DB 保存は別モードとして後付け対応可能
